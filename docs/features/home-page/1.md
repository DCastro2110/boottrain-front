# Task 1: Create ConsistencyBoard component

## Description

Create a `ConsistencyBoard` client component that renders 7 day squares representing the current week's exercise consistency. Each square's color indicates the user's exercise status for that day. This component receives data via props from the parent (server) page component.

## Files to Create

- `src/components/ConsistencyBoard.tsx`

## Implementation Details

1. Create a client component (`"use client"` directive)
2. Props interface:
   ```typescript
   interface IConsistencyBoardProps {
     consistency: Array<{
       day: 'SUNDAY' | 'MONDAY' | 'TUESDAY' | 'WEDNESDAY' | 'THURSDAY' | 'FRIDAY' | 'SATURDAY';
       status: 'completed' | 'not_completed' | 'missed';
     }>;
     fireSequence: number;
   }
   ```
3. Render 7 squares in a horizontal flex layout with 12px gap
4. Color mapping:
   - `completed` → Blue (`bg-blue-500` equivalent)
   - `not_completed` → White/Light (`bg-neutral-100` equivalent)
   - `missed` → Light blue (`bg-blue-100` equivalent)
5. Each square should display the day abbreviation (DOM, SEG, TER, QUA, QUI, SEX, SÁB)
6. Display fire sequence badge with orange accent showing the streak number
7. Use Tailwind CSS classes - no hardcoded colors

## Acceptance Criteria

- [ ] Component is a client component at `src/components/ConsistencyBoard.tsx`
- [ ] Props interface is properly typed
- [ ] Renders exactly 7 day squares (Sunday to Saturday)
- [ ] Squares use correct colors based on status:
  - Blue for completed
  - Light blue for started/missed
  - White for not started
- [ ] Day abbreviations are displayed (DOM, SEG, TER, QUA, QUI, SEX, SÁB)
- [ ] Fire sequence number is displayed with orange accent
- [ ] Component can be imported from `@/components/ConsistencyBoard`

## Dependencies

- None (pure presentational component, receives data via props)
