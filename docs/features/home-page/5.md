# Task 5: Create TreinoDeHoje section component

## Description

Create a `TreinoDeHoje` client component that conditionally renders either the `WorkoutCard` or `RestDayCard` based on whether `todayWorkoutDay` data exists. This component receives data via props from the parent (server) page component.

## Files to Create

- `src/components/TreinoDeHoje.tsx`

## Implementation Details

1. Create a client component (`"use client"` directive)
2. Props interface:
   ```typescript
   interface ITreinoDeHojeProps {
     todayWorkoutDay: {
       date: string;
       name: string;
       estimatedDurationInSeconds: number;
       numberOfExercises: number;
       coverImageUrl: string | null;
       isCompleted: boolean;
     } | null;
   }
   ```
3. Render section with:
   - Section title "Treino de Hoje" (18px, weight 600)
   - "Ver treinos" link (12px, blue) pointing to workout history
   - Conditionally render WorkoutCard (if todayWorkoutDay exists) or RestDayCard (if null)
4. For RestDayCard, determine the current weekday name from the date
5. Use flexbox vertical layout with 12px gap between elements

## Acceptance Criteria

- [ ] Component is a client component at `src/components/TreinoDeHoje.tsx`
- [ ] Props interface properly typed with todayWorkoutDay (nullable)
- [ ] Displays "Treino de Hoje" title
- [ ] Displays "Ver treinos" link
- [ ] Conditionally renders WorkoutCard when todayWorkoutDay is not null
- [ ] Conditionally renders RestDayCard when todayWorkoutDay is null
- [ ] Passes correct weekday to RestDayCard
- [ ] Component can be imported from `@/components/TreinoDeHoje`

## Dependencies

- Task 2 (WorkoutCard component)
- Task 3 (RestDayCard component)

## Notes

- This is a thin wrapper that handles the conditional logic
- Weekday name mapping for Portuguese:
  - SUNDAY → Domingo
  - MONDAY → Segunda
  - TUESDAY → Terça
  - WEDNESDAY → Quarta
  - THURSDAY → Quinta
  - FRIDAY → Sexta
