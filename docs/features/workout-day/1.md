# Task 1: Setup Routing, Authentication, and Authorization Handling

## Description
Create the base route for the Workout Day page and implement the required authentication and authorization guards.

## Acceptance Criteria
- [ ] Create the Next.js page at `src/app/workout-plan/[id]/workout-day/[workoutDayId]/page.tsx`.
- [ ] Implement an authentication check. If the user is not logged in, redirect them to `/login`.
- [ ] Implement an authorization boundary. If the user lacks permission to view the page, display a specific UI with the message "Você não tem permissão para acessar esta página.".
- [ ] Include a "Voltar para o menu" button in the unauthorized view that navigates the user back to the home page (`/`).
