# Contexto
A página do plano precisa buscar os dados reais da API e tratar os casos em que o usuário não tem permissão para visualizar o conteúdo (acessando um plano que não pertence a ele).

# Objetivo
Fazer o fetch da API para o plano de treino específico e implementar uma "Error State View" para tratar faltas de permissão, além do estado de loading dos dados.

# Arquivos Afetados
- `src/app/planos/[planId]/page.tsx`

# Instruções de Implementação
1. No componente da página, importe `getWorkoutPlanWorkoutPlanId` (ou função similar gerada no seu `boo-train-api.ts`).
2. Crie estados locais usando `useState`: `planData` (para armazenar a resposta da API), `isFetchingPlan` (boolean para controle de loading), e `fetchError` (para armazenar erro).
3. Utilize `useEffect` para iniciar o fetch assim que o componente montar e tiver o `planId`.
4. Trate a resposta da API: se der sucesso, grave em `planData`.
5. Se a promessa da API for rejeitada ou retornar `status: 401` ou `status: 403` (verifique o shape do retorno do Orval que muitas vezes retorna os dados do erro):
   - Ajuste o estado de `fetchError` com `true`.
6. Na função de renderização, adicione a lógica de permissão negada:
   - Se houver `fetchError` (e representar falta de permissão):
   - Renderize uma tela amigável, contendo EXATAMENTE o texto: "Você não tem permissão para acessar esta página."
   - Importe e inclua o botão do sistema (ex: de `src/components/ui/button.tsx`) com a label "Voltar para o menu", que ao ser clicado chama `router.push('/')`.
   - Essa tela deve ser centralizada e limpa.

# Critérios de Aceite
- Ao acessar a página, um loading deve ocorrer enquanto a API é chamada.
- Se a API retornar sucesso, os dados devem estar salvos no state.
- Em caso de acesso não autorizado, a exata string "Você não tem permissão para acessar esta página." deve ser mostrada com o botão "Voltar para o menu".
