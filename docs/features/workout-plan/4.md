# Contexto
Com os dados do plano disponíveis no estado da página, precisamos renderizar as informações visuais (Header e os cards dos dias) respeitando os designs (Node ID: qW6Lu para a tela e a3W9s para cards) e proporcionando responsividade.

# Objetivo
Construir a interface final renderizando o Header com os dados do plano e a lista de Workout Days em formato de cards interativos reutilizáveis, com redirecionamento de links.

# Arquivos Afetados
- `src/app/planos/[planId]/page.tsx`

# Instruções de Implementação
1. **Header**: 
   - Quando `planData` estiver preenchido, renderize no topo um cabeçalho contendo o `planData.name` como título principal (`<h1>` estilizado, ex: `text-2xl font-bold`) e `planData.description` como um sub-texto.
2. **Cards**: 
   - Importe o componente de card utilizado para dias de treino na home page (ex: `WorkoutCard` ou `RestDayCard` ou equivalente em `src/app/home/components/`).
   - Mapeie (`planData.workoutDays.map`) o array de dias.
   - Envolva cada card renderizado em um componente `Link` do Next.js (ou use `onClick` com `router.push`) apontando para `/planos/[planId]/dias/[dayId]` onde `[dayId]` é o id iterado de cada `workoutDay`.
   - Passe os dados para o card correspondente conforme esperado por suas props (ex: nome do dia, duração estimada, exercícios ou isRestDay).
3. **Responsividade**:
   - Envolva a lista de cards em um container CSS Grid (`<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">`). Isso garante o comportamento fluído entre mobile e desktop.

# Critérios de Aceite
- O nome e descrição do plano carregado devem ser visíveis no topo da tela.
- Os cards dos dias de treino devem exibir as informações corretamente usando os mesmos componentes visuais da home page.
- Ao clicar no card, o aplicativo deve navegar para a tela de detalhes do dia.
- A página deve manter-se coesa, amigável e completamente responsiva do celular ao desktop.
