# Contexto

Precisamos criar a página raiz do plano de treino (`/planos/[planId]`) e garantir que apenas usuários logados tenham acesso a ela.

# Objetivo

Criar a página principal em `/planos/[planId]` com uma estrutura inicial de loading/container e implementar a proteção de rota (redirecionamento para `/login`).

# Arquivos Afetados

- `src/app/planos/[planId]/page.tsx` (novo)

# Instruções de Implementação

1. Crie o arquivo `src/app/planos/[planId]/page.tsx` e defina um componente funcional React padrão exportado por default.
2. O componente deve extrair o parâmetro da rota através da prop de página `params` no formato `Promise<{ planId: string }> | { planId: string }` de acordo com a versão do Next.js App Router (usualmente `const planId = (await params).planId` em Server Components ou `use()` em Client Components, porém como usaremos hooks de contexto, é mais fácil fazê-la um `"use client"` component onde `const { planId } = use(params)` ou receber `params` normal).
3. Importe e utilize o hook `useUser` (de `src/hooks/use-user.ts`) para checar se o usuário está logado.
4. Implemente a lógica: se `isLoading` do `useUser` for false e não houver um usuário válido logado, utilize o `useRouter` (de `next/navigation`) para redirecionar o visitante imediatamente para `/login`.
5. Se estiver em carregamento do `useUser`, mostre um indicador de carregamento básico (ex: `<div className="flex justify-center p-8">Carregando...</div>`).
6. Retorne um container div para envolver o conteúdo futuro (ex: `<div className="container mx-auto p-4 md:p-8 space-y-6"></div>`).

# Critérios de Aceite

- A página `/planos/[planId]` deve renderizar e compilar sem erros.
- Se um usuário anônimo (não autenticado) acessar a rota, o comportamento deve ser um redirecionamento imediato para `/login`.
- A página atua como Client Component (`"use client"`) caso necessário para funcionar perfeitamente com `useUser` e `useRouter`.
