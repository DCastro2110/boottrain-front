# Contexto

A funcionalidade de Workout Plan deve usar URLs em português. Atualmente, existe uma rota parcial em inglês `src/app/workout-plan/[id]/workout-day/[workoutDayId]/page.tsx` (e seus componentes em `components/hero.tsx`) que precisa ser movida para a nova estrutura. Além disso, todos os diretórios de componentes locais em rotas devem seguir a convenção de nomeação `_components`.

# Objetivo

Renomear os diretórios para seguir a nova estrutura de rotas em português `/planos/[planId]/dias/[dayId]` e garantir que os componentes locais utilizem a pasta `_components`.

# Arquivos Afetados

- `src/app/workout-plan/` (diretório a ser movido/removido)
- `src/app/planos/[planId]/dias/[dayId]/page.tsx` (novo destino da página)
- `src/app/planos/[planId]/dias/[dayId]/_components/hero.tsx` (novo destino dos componentes locais)

# Instruções de Implementação

1. Crie a nova estrutura de diretórios base: `src/app/planos/[planId]/dias/[dayId]/` e `src/app/planos/[planId]/dias/[dayId]/_components/`.
2. Mova o arquivo `page.tsx` de `src/app/workout-plan/[id]/workout-day/[workoutDayId]/page.tsx` para a nova pasta `src/app/planos/[planId]/dias/[dayId]/page.tsx`.
3. Mova os componentes locais de `src/app/workout-plan/[id]/workout-day/[workoutDayId]/components/hero.tsx` para `src/app/planos/[planId]/dias/[dayId]/_components/hero.tsx`.
4. Abra o arquivo `page.tsx` migrado e atualize o import do componente `hero.tsx` para refletir o novo caminho relativo `./_components/hero`.
5. Verifique e corrija quaisquer outros imports (hooks, API, etc) que possam ter quebrado com a mudança de diretórios.
6. Após validar a migração, exclua completamente a pasta antiga `src/app/workout-plan`.

# Critérios de Aceite

- A pasta `src/app/workout-plan` não deve mais existir no projeto.
- A página de detalhes do dia de treino deve estar em `src/app/planos/[planId]/dias/[dayId]/page.tsx`.
- Os componentes locais da página do dia (como o `hero.tsx`) devem estar dentro da pasta `_components` (`.../dias/[dayId]/_components/hero.tsx`).
- O projeto e a página do dia devem compilar corretamente após as mudanças de imports.
