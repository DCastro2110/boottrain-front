# Task 3: Implement Google OAuth Sign-In Handler

## Objective

Implement the actual Google OAuth authentication flow using better-auth.

## Deliverables

- Updated `GoogleAuthButton` component with proper sign-in implementation
- Error handling for failed authentication attempts

## Requirements

### Functional Requirements

- Call `signIn("google")` from better-auth/react on button click
- Redirect to Google OAuth consent screen
- Handle successful authentication (redirect to authenticated area)
- Handle authentication failures gracefully

### Technical Requirements

- Import: `import { signIn } from "better-auth/react"`
- On click handler: `await signIn("google")`
- Wrap in try-catch for error handling
- Set loading state during the auth process
- Clear loading state on completion (success or failure)

### Error Handling

| Error Type | User Message | Action |
|------------|--------------|--------|
| Network error | "Erro de conexão. Tente novamente." | Show retry button |
| Auth cancelled | "Login cancelado. Tente novamente." | Reset to default state |
| Unknown error | "Algo deu errado. Tente novamente." | Show error, allow retry |

### Acceptance Criteria

- [ ] Clicking button initiates Google OAuth flow
- [ ] Successful auth redirects user appropriately
- [ ] Network errors display appropriate message
- [ ] User can retry after errors
- [ ] Loading state shown during auth
- [ ] No console errors during auth flow
