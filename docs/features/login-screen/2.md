# Task 2: Create GoogleAuthButton Client Component

## Objective

Create the interactive Google sign-in button component that handles the OAuth flow when clicked.

## Deliverables

- `src/components/auth/GoogleAuthButton.tsx` - Client component with sign-in functionality

## Requirements

### Functional Requirements

- Display a white pill-shaped button with:
  - Google icon (16x16)
  - Text: "Fazer login com Google" (Portuguese)
- On click, trigger Google OAuth sign-in via better-auth
- Handle loading state while authentication in progress
- Handle error state and display error messages

### Technical Requirements

- Use `"use client"` directive
- Import `signIn` from `better-auth/react`
- Use shadcn/ui Button component for base styling
- Button dimensions: 38px height, pill shape (border-radius: 100px)
- Gap between icon and text: 8px
- Button text: Inter font, 14px, weight 600, black (#000000)

### Component Interface

```typescript
interface IGoogleAuthButtonProps {
  // No props required for initial implementation
  // Can be extended for callback URL, etc.
}
```

### States

| State | Visual |
|-------|--------|
| Default | White button, black text, Google icon |
| Hover | Platform-standard hover effect |
| Loading | Button disabled, loading indicator |
| Error | Error message displayed, button re-enabled |

### Acceptance Criteria

- [ ] Component renders button with correct styling
- [ ] Button text is "Fazer login com Google" in Portuguese
- [ ] Button has pill shape (full rounded corners)
- [ ] Google icon visible next to text
- [ ] Click triggers signIn function
- [ ] Loading state disables button
- [ ] Error state shows message
