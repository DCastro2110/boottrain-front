# Task 5: Add Error Handling and Loading States

## Objective

Implement comprehensive error handling and loading states for the login flow.

## Deliverables

- Updated `GoogleAuthButton` component with robust error handling
- Loading spinner/indicator during authentication
- Error messages displayed to user in Portuguese

## Requirements

### Loading State

- Button disabled during authentication
- Show loading indicator (spinner or text change)
- Text changes to "Entrando..." or similar during loading

### Error States

| Scenario               | Message                                         | Recovery          |
| ---------------------- | ----------------------------------------------- | ----------------- |
| Network failure        | "Erro de conexão. Verifique sua internet."      | Retry button      |
| Auth cancelled by user | "Login cancelado pelo usuário."                 | Return to default |
| Server error           | "Erro no servidor. Tente novamente mais tarde." | Retry button      |
| Unknown error          | "Algo inesperado ocorreu. Tente novamente."     | Retry button      |

### Error Display

- Show error message below the button
- Error text: red-ish color, smaller font
- Clear error when user initiates new attempt
- Accessible (sr-only for screen readers if needed)

### Accessibility

- Loading state announced to screen readers
- Error messages associated with button via aria-describedby
- Focus management: return focus to button after error

### Acceptance Criteria

- [ ] Loading state shows spinner/text
- [ ] Button disabled during loading
- [ ] Network errors display specific message
- [ ] Auth cancellation handled gracefully
- [ ] Error messages in Portuguese
- [ ] User can retry after any error
- [ ] Accessibility: proper ARIA attributes
- [ ] No unhandled promise rejections
