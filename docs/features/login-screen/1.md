# Task 1: Create Login Page Route and Server Component

## Objective

Create the login page route at `/login` with a server component that renders the basic page structure, branding, and contains the interactive Google auth button.

## Deliverables

- `src/app/login/page.tsx` - Main login page server component
- `src/app/login/layout.tsx` (optional) - Layout if needed for the login route

## Requirements

### Functional Requirements

- Page route: `/login`
- Render Fit.ai logo/branding at top
- Render blue gradient card section with hero message
- Include `GoogleAuthButton` client component for interactivity
- Handle redirect for already-authenticated users (via middleware or component logic)

### Technical Requirements

- Use Next.js App Router (server component by default)
- Import `GoogleAuthButton` from `@/components/auth/GoogleAuthButton`
- Apply styling matching Pencil design (dark background, blue card)
- Use Tailwind classes, no hardcoded colors
- Follow component colocation: if only used in login page, can define in `page.tsx`

### Acceptance Criteria

- [ ] Page accessible at `/login` route
- [ ] Page renders without errors
- [ ] Logo/branding visible at top
- [ ] Blue card with hero text visible
- [ ] GoogleAuthButton rendered within the card
- [ ] No TypeScript errors
