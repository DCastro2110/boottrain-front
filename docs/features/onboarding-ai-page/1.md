# Task 1: Create onboarding page route structure

## Description

Create the directory and page file for `/onboarding` route. This will be the entry point for users who need to complete their profile before accessing the app.

## Requirements

- Create `src/app/onboarding/page.tsx` as a server component
- The page should check authentication (redirect to /login if not authenticated)
- Import and use existing AI components from `@/components/ai`
- The page should display the AI chat interface for onboarding

## Acceptance Criteria

- File exists at `src/app/onboarding/page.tsx`
- Page is a server component that checks authentication
- Unauthenticated users are redirected to `/login`
- Uses existing AI components for the chat interface

## Notes

- Based on design OY1JW, the onboarding is a full-screen chat experience
- The page should have the "Coach AI" header with online indicator
- Welcome messages should be displayed in gray chat bubbles
- A "Começar" button should be shown to start the conversation
