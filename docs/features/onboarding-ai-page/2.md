# Task 2: Implement profile completeness validation proxy

## Description

Create `src/proxy.ts` with a function to check if user has all required profile fields (weight, height, age, body fat percentage). This is a Next.js utility that validates profile completeness and handles redirects.

## Requirements

- Create function `hasUserCompletedOnboarding(userId)` that returns boolean
- Check that height, weight, age, bodyFatPercentage are all non-null
- Use `getUsersUserId` API call from `@/lib/api/boo-train-api.ts` to fetch user profile

## Acceptance Criteria

- Function `hasUserCompletedOnboarding(userId)` exists and returns boolean
- Checks that all four fields (height, weight, age, bodyFatPercentage) are non-null
- Uses the generated API client from `@/lib/api/booTrainAPI`
- Returns `true` if all fields present, `false` otherwise

## Notes

- This proxy will be used by the onboarding page and protected routes
- The validation logic is: `height !== null && weight !== null && age !== null && bodyFatPercentage !== null`
