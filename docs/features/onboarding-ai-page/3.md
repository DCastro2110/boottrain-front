# Task 3: Restrict onboarding page access to incomplete profiles only

## Description

The onboarding page itself should validate whether the user has a complete profile. If the user already has all required fields (weight, height, age, body fat percentage), they should be redirected away from the onboarding page (to home). This keeps login and home pages unchanged.

## Requirements

- Onboarding page (`src/app/onboarding/page.tsx`) validates profile on load
- User with complete profile → redirect to home page
- User with incomplete profile → access onboarding page
- No modifications to login page or home page

## Acceptance Criteria

- Onboarding page checks profile completeness on render
- Users who completed onboarding are redirected to home
- Users who need onboarding can access the page
- Login and home pages remain unchanged

## Notes

- The check happens inside the onboarding page component
- Use `redirect` from `next/navigation` for server-side redirects
- If user has all four fields (height, weight, age, bodyFatPercentage), redirect to `/`
