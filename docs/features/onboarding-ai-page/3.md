# Task 3: Add redirect logic to protected routes

## Description

Update home page and other protected routes to redirect to `/onboarding` if user profile is incomplete.

## Requirements

- Home page (`src/app/(home)/page.tsx`) should check onboarding status
- User without complete profile should be redirected to `/onboarding`
- Login page should also check and redirect appropriately

## Acceptance Criteria

- Home page checks onboarding status on load
- Incomplete profiles redirect to `/onboarding` before rendering home
- Login page checks if user already completed onboarding (skip if complete)
- The redirect logic uses the `hasUserCompletedOnboarding` proxy

## Notes

- The redirect should happen server-side when possible
- For client-side checks, use `useEffect` with router.push
- Consider creating a shared hook or utility for reusable redirect logic
